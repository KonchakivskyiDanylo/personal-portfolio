Ukrainian Air Alert System

University ML project for air raid alert prediction

The Assignment

This was a university project at NaUKMA where our task was to create a 24-hour air raid alert prediction system. The assignment required us to work with multiple data sources and build a complete machine learning pipeline from data collection to user interface.

Data Sources

  • Historical Alert Data: 3 years of air raid alert records
  • Weather Data: Hourly meteorological information
  • News Reports: Daily reports processed with TF-IDF

Project Stages

  • Parse and collect historical data
  • Perform data analysis with visualizations
  • Train and evaluate ML models
  • Create user interface for predictions

Our Solution

We approached this project with a focus on automation and user accessibility. While the core ML task was challenging, we made strategic decisions to create a practical, user-friendly system.

Data Pipeline

Automated hourly data collection using cron jobs to continuously gather weather data and process daily news reports for real-time predictions.

ML Approach

Tested multiple algorithms and selected Random Forest for our final model, focusing on practical performance over theoretical perfection.

Smart Interface Choice

Created a simple web interface with Ukraine map, but made Telegram bot our primary interface since it's better for real-time notifications.

Telegram Bot - Our Main Innovation

Recognizing that people use Telegram more than websites for notifications, we built a comprehensive bot that became the heart of our system.

/predict Command

Users select their region and receive a 24-hour prediction for air raid alerts in their area.

/alarms Command

Shows current active air raid alerts across different regions for immediate situational awareness.

Location-Based Notifications

When users register, their location is saved to MongoDB. They automatically receive notifications when alerts begin or end in their region.

Scheduled Daily Predictions

Users choose their preferred time (e.g., 12:00) and receive automated daily 24-hour predictions via cron-scheduled messages.

Technical Implementation

System Architecture

Built on AWS EC2 with Flask backend, MongoDB for data storage, and integrated Telegram API for user interaction. The system runs continuously with automated data collection and processing.

Data Processing Pipeline
  • Hourly cron jobs collect weather data for next 24 hours
  • Daily news reports processed through TF-IDF vectorization
  • Historical alert patterns analyzed for model training
  • MongoDB stores user data, predictions, and historical records
ML & Automation
  • Random Forest model selected after testing multiple algorithms
  • Flask REST API serves predictions and data visualization
  • Cron scheduling for automated daily predictions
  • Real-time alert monitoring and user notifications

Technologies Used

Backend: Python Flask MongoDB
ML & Data: Scikit-learn Pandas MatPlotlib TF-IDF Random Forest
Infrastructure: AWS EC2 Telegram API Cron Jobs

Reflection & Learning

Technical Value

This project provided excellent hands-on experience with real-time data pipelines, automated systems, cloud deployment, and team collaboration on ML projects. The integration of multiple technologies and services was particularly valuable.

Realistic Assessment

Honestly, predicting human strategic decisions through weather patterns and news reports is fundamentally impossible. Air raid alerts depend on tactical, political, and strategic factors that don't correlate with environmental data. The model accuracy metrics were misleading since "no alert" is the most frequent outcome.
