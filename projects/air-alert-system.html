Ukrainian Air Alert System

Real-time air alert monitoring system for Ukraine

Project Context

This was a group project for my coursework at NaUKMA (National University of Kyiv-Mohyla Academy). All groups received the same assignment: create an air raid alert prediction system using weather data and daily news reports.

The Assignment

Our task was to build a prediction system that could forecast air raid alerts using two main data sources:

  • Weather Data: Hourly meteorological information for pattern analysis
  • Daily News Reports: Text analysis of daily news summaries

The system needed to provide 24-hour predictions and include a user-friendly interface for interaction.

Our Approach

Data Collection & Processing

  • Weather Data: Automated hourly collection using cron jobs
  • News Reports: Daily text data for natural language processing
  • Air Alert Records: Historical alert data from public sources
  • Database: MongoDB for flexible storage of weather, text, and user data

Machine Learning & NLP Pipeline

  • Weather Analysis: Time-series modeling with multiple algorithms
  • Text Processing: NLP pipeline including stemming, lemmatization, and TF-IDF vectorization
  • Model Training: GridSearchCV for hyperparameter optimization
  • Feature Engineering: Combined weather and text features

System Architecture

  • Flask Web Application: REST API and web interface
  • Telegram Bot API: User interaction and notification system
  • MongoDB: User data, predictions, and historical records
  • AWS EC2: Cloud hosting and processing
  • Cron Jobs: Data collection and daily prediction scheduling

Data Analysis & Visualization

  • Jupyter Notebooks: Comprehensive analysis of historical alert patterns
  • Weather Correlation Analysis: Statistical analysis of weather-alert relationships
  • Text Analysis: News sentiment and keyword frequency analysis
  • Temporal Patterns: Time-based alert frequency and duration analysis

Telegram Bot Interface

To make the system more accessible and user-friendly, we developed a Telegram bot that serves as the main interface for users.

/predict Command

Users can select their region and receive 24-hour predictions for air raid alerts in their area.

/alarms Command

Displays current active air raid alerts across different regions.

Location-Based Notifications

When users start the bot, their location is saved to MongoDB. They automatically receive notifications when alerts begin or end in their region.

Daily Predictions

Users can set their preferred time to receive daily 24-hour predictions via cron-scheduled messages.

Results

~80% Best Model Accuracy

Key Insights & Reflections

Limitations of Weather-Based Prediction

While we achieved decent statistical performance, the project revealed fundamental limitations in predicting human conflict behavior through environmental data alone. Air raid alerts depend heavily on strategic, political, and tactical decisions that don't correlate strongly with weather patterns.

Technical Learning Experience

Despite the predictive limitations, this project provided valuable experience with:

  • Real-time data collection and processing pipelines
  • Working with time-series data and temporal patterns
  • Cloud deployment and automation
  • Team collaboration on ML projects

Academic Value

This project served as an excellent learning exercise in applying ML to real-world problems, understanding data limitations, and recognizing when certain problems may not be well-suited for pure data-driven approaches. It taught us the importance of domain knowledge and the complexity of human behavior prediction.
